# Maths OCC

``Maths OCC`` (French for Objets ConnectГ©s & CybersecuritГ©), is an interactive web application developed as part of the mathematics curriculum for the first year of a ``Master's degree programme in Artificial Intelligence``, with a specialisation in ``IoT Devices and Cybersecurity``. **Backend** вҖ” Python 3.11+, Flask. No external database; state is persisted in a local `data.json` file via a lightweight JSON store abstraction.

**Frontend** вҖ” Pure HTML, CSS, and vanilla JavaScript. No frontend framework or build step required. Typography uses the Geist and Geist Mono typefaces. The UI supports dark and light themes, is responsive down to mobile viewports, and is fully bilingual (French / English) with live language switching without page reload.

**Internationalisation** вҖ” all user-facing strings generated by the Python backend (step-by-step traces, verdicts, error messages) are passed through a central `core/i18n.py` translation layer keyed on a `lang` parameter sent with every API request. Frontend strings are handled by a JavaScript `TR` dictionary and a single `applyLang()` function.

## Features & Modules

The application is organised into seven functional tabs, each corresponding to a distinct mathematical domain.

### 1. Set Operations (`/api/ops`)
Implements the six fundamental operations of naive set theory on integer sets provided by the user:

- **Union** A вҲӘ B
- **Intersection** A вҲ© B
- **Difference** A \ B
- **Symmetric Difference** A в–і B
- **Cartesian Product** A Г— B
- **Cardinality** |A|

All operations are performed on parsed integer sets and results are logged to a persistent history store.

### 2. Set-Builder Notation (`/api/comprehension`)
Allows the construction of a set by comprehension over an integer universe E = [start, end], filtering elements using a Python predicate expression P(x). Formally: A = { x вҲҲ E | P(x) }.

### 3. Functions & Applications (`/api/function`)
Given a domain A, codomain B, and a formula f(x) expressed as a Python expression, this module:

- Builds the function table f : A вҶ’ B
- Computes the **direct image** f(S) for a subset S вҠҶ A
- Computes the **inverse image** fвҒ»В№(T) for a subset T вҠҶ B
- Determines whether f is **injective**, **surjective**, and/or **bijective**

### 4. Saved Sets (`/api/sets`)
A named persistent store allowing the user to save, load, and delete integer sets by name. Saved sets can be loaded directly into the Operations tab for reuse across computations.

### 5. Relations (`/api/relations`)
Covers two families of binary relations with full proof-level detail:

**Order Relations** вҖ” verifies the three conditions (reflexivity, antisymmetry, transitivity) and determines whether the order is total or partial. Implemented relations include:
- вүӨ on в„Ө (total order)
- < on в„Ө (strict total order)
- Divisibility | on в„•* (partial order)
- Inclusion вҠҶ on рқ’«(E) (partial order)

**Equivalence Relations** вҖ” verifies reflexivity, symmetry, and transitivity, and describes the resulting equivalence classes. Implemented relations include equality, congruence mod n, same parity, same sign, line parallelism, and set equipotence.

**Congruence mod n** (`/api/relations/modulo`) вҖ” given integers a, b, and modulus n вүҘ 2, checks whether a вүЎ b (mod n) with a step-by-step arithmetic proof.

### 6. Prime Numbers (`/api/prime`)
Two sub-modes for primality analysis:

- **Single check** вҖ” trial division up to вҲҡn with a full verbose step-by-step trace, divisor identification, and prime factorisation for composite numbers.
- **Range check** вҖ” GCD-based primality test (verifying gcd(n, k) = 1 for all k вҲҲ [2, вҲҡn]) applied to every integer in an interval [a, b], with an optional verbose output. Limited to intervals of at most 10,000 values.

### 7. Cryptography (`/api/crypto`)
Three educational cryptographic tools directly relevant to the cybersecurity specialisation:

**Base64** вҖ” encodes and decodes strings through the full bit-level pipeline: UTF-8 byte conversion вҶ’ binary representation вҶ’ 6-bit grouping вҶ’ Base64 alphabet mapping вҶ’ padding.

**Caesar Cipher** вҖ” a classical monoalphabetic substitution cipher. Supports encoding, decoding, and full brute-force attack (all 25 shifts), with character-by-character trace output.

**RSA** вҖ” an educational implementation of the RSA asymmetric cryptosystem. Covers:
- Prime generation via the Miller-Rabin probabilistic primality test
- Key generation: n = pГ—q, ПҶ(n) = (pвҲ’1)(qвҲ’1), selection of e, computation of d = eвҒ»В№ mod ПҶ(n) via the Extended Euclidean Algorithm
- Message encryption: c = mбөү mod n
- Message decryption: m = cбөҲ mod n

> вҡ пёҸ The RSA implementation is strictly educational and must not be used in production contexts. Key sizes are limited to 16вҖ“256 bits.

### 8. History (`/api/history`)
A chronological log of all set operations performed during the session, with timestamps, operands, and results. The history is persistent across sessions and can be cleared on demand.

## Mathematical Foundations

The following mathematical structures and results are either implemented or referenced within the project:

- **Naive set theory** вҖ” operations, comprehension, power sets, Cartesian products
- **Functions** вҖ” injectivity, surjectivity, bijectivity, direct and inverse images
- **Binary relations** вҖ” reflexivity, symmetry, antisymmetry, transitivity; order and equivalence relations; quotient sets
- **Modular arithmetic** вҖ” congruence classes, в„Ө/nв„Ө, the ring of integers modulo n
- **Elementary number theory** вҖ” prime numbers, trial division, GCD, the Euclidean and Extended Euclidean algorithms, Euler's totient function ПҶ
- **Probabilistic primality testing** вҖ” Miller-Rabin witness rounds
- **Asymmetric cryptography** вҖ” RSA key generation, modular exponentiation, the relationship between hard number-theoretic problems and cryptographic security
